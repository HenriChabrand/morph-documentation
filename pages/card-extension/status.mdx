# Status

Status content provides a way to succinctly show a user the status of a certain operation or process. The `CardStatus` integrated into our API defines the following states: 'DEFAULT', 'INFO', 'SUCCESS', 'WARNING', and 'DANGER'.

## API Reference

### CardStatus

A part of a card that can show status according to context-specific condition.

Example:

```tsx
  const newCard = cardBuilder.newCard('Services');

  newCard.newStatus('DB Service', 'Available', 'SUCCESS');
  newCard.newStatus('API Service', 'Degraded Performance', 'WARNING');
  newCard.newStatus('Logging Service', 'Unavailable', 'DANGER');
```

## Props

Prop  |   Description   |  Type  |  Default
----- | --------------- | ------ | ---------
name*  | The name of status field, displayed as the label.   |   string | -
value* | The value of the status field to display next to the label. |   string  |   -
type  | The visual style to apply depending on the status. |  'DEFAULT' \| 'INFO' \| 'SUCCESS' \| 'WARNING' \| 'DANGER'  | 'DEFAULT'

`*` indicates required fields
